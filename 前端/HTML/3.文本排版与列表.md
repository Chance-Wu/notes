>学习目标
>
>- 精准掌握标题标签（h1-h6）的层级规范与 SEO 作用
>- 理解语义化文本标签（strong/em）与普通标签的本质区别
>- 熟练运用无序列表（ul/li）和有序列表（ol/li）构建内容结构
>- 创建符合语义规范的个人简介页面



### 一、标题标签的层级规范

---

#### 1.1 层级逻辑（必需严格遵循）

| 标签          | 作用       | 使用场景                  | SEO权重 |
| ------------- | ---------- | ------------------------- | ------- |
| `<h1>`        | 页面主标题 | 仅允许1个（如"个人简介"） | 最高    |
| `<h2>`        | 一级子标题 | 主要章节（如"教育经历"）  | 次高    |
| `<h3>`        | 二级子标题 | 子章节（如"大学经历"）    | 中      |
| `<h4>`-`<h6>` | 三级及以上 | 细节说明                  | 低      |

#### 1.2 关键规范

- 正确：`<h1>个人简介` → `<h2>教育背景` → `<h3>大学经历`
- 错误：`<h1>个人简介` → `<h3>教育背景`（跳过h2层级）
- 错误：多个`<h1>`（如每个章节都用h1）

#### 1.3 SEO 原理

搜索引擎通过标题层级判断内容重要性，h1 是页面主题标识。



### 二、语义化文本标签深度解析

---

#### 2.1 层级逻辑（必需严格遵循）

| 标签       | 语义含义   | 作用          | 浏览器默认样式 | 正确使用场景                         |
| ---------- | ---------- | ------------- | -------------- | ------------------------------------ |
| `<strong>` | 强调重要性 | 标记关键信息  | 加粗           | "核心技能：`<strong>React</strong>`" |
| `<em>`     | 强调语气   | 表达情感/重点 | 斜体           | "我`<em>非常</em>`喜欢前端开发"      |
| `<span>`   | 无语义容器 | 仅用于样式    | 无变化         | `style="color:blue"`（非内容语义）   |

####  2.2 核心区别

- 错误：`<span style="font-weight:bold">核心技能</span>`（用样式模拟语义）
- 正确：`<strong>核心技能</strong>`（用语义标签表达重要性）
- 为什么重要：屏幕阅读器会读出`<strong>`为"重要"，`<em>`为"强调"



### 三、列表标签的语义规范

---

#### 3.1 无序列表（ul/li）

无序列表（unordered list）、列表项（list item）

- 用于**无顺序**的项目（如技能、特点）
- 语法：`<ul><li>项目1</li><li>项目2</li></ul>`
- **默认样式：圆点**（可自定义：`<ul style="list-style-type: square">`）

```html
<!DOCTYPE html>
<html>

<head>
    <meta charset="utf-8">
    <title>文本排版与列表</title>
</head>

<body>
    <h1>个人简介</h1>
    <h2>教育经历</h2>
    <h3>大学经历</h3>

    2010-2011 <strong>学生会</strong>
    <br>
    2011-2012 <span>计算机科学与技术</span>
    <br>
    我<em>特别</em>喜欢听音乐

    <h4>技能</h4>
    <!-- 无序列表 -->
    <ul style="list-style:square">
        <li>HTML</li>
        <li>CSS</li>
        <li>JavaScript</li>
    </ul>

</body>

</html>
```

#### 3.2 有序列表（ol/li）

有序列表（ordered list）、列表项（list item）

- 用于**有顺序**的内容（如步骤、时间线）
- 语法：`<ol><li>步骤1</li><li>步骤2</li></ol>`
- **默认类型：数字**（可改：`<ol type="A">`→A/B/C）

---

```html
<!DOCTYPE html>

</head>

                <li>Javascript (ES6+）</li>
                <li>响应式设计（Mobile First）</li>

            </ol>
        </article>
    </main>

        <p>©️ 2025 chance · 个人作品集</p>
    </footer>

</body>

</html>
```



### 五、关键建议

---

#### 5.1  标题层级必须连贯

- 错误：`<h1>简介` → `<h3>技能`（跳过h2）
- 正确：`<h1>简介` → `<h2>技能` → `<h3>React`

#### 5.2 避免标签滥用

- 错误：用`<h2>`做按钮标题（应改用`<button>`）
- 错误：用`<strong>`包裹所有文字（仅关键信息）

#### 5.3 列表的语义选择

- 技能/特点 → **无序列表**（ul）
- 步骤/时间线 → **有序列表**（ol）
- 例：简历中的"工作经历"用ol（按时间顺序）
