### 一、图像标签 `<img>` 深度解析

---

#### 1.1 必须属性

| 属性 | 作用     | 规范要求               | 示例             |
| ---- | -------- | ---------------------- | ---------------- |
| src  | 图片路径 | 必须提供有效 URL       | src="avatar.jpg" |
| alt  | 替代文本 | 必须填写（无障碍核心） | alt="个人头像"   |

#### 1.2 关键原则

- alt 是语义中心：

  - 屏幕阅读器会朗读 alt 内容
  - 图片加载失败时显示替代文本
  - SEO 优化：搜索引擎通过 alt 理解图片内容

- 错误示例：

  ```html
  <!-- 无alt（严重无障碍问题） -->
  <img src="photo.jpg">
  
  <!-- alt为空但非装饰图 -->
  <img src="logo.png" alt="">r
  ```

#### 1.3 响应式图片进阶

```html
<!-- 基础响应式（推荐第5天掌握） -->
<img src="image.jpg" 
     alt="描述性文字"
     width="300" 
     height="200"
     style="max-width: 100%; height: auto;">
```

- `width`/`height`：防止布局偏移（CLS优化）
- `max-width: 100%`：确保图片不超出容器



### 二、超链接 `<a>` 核心规范

---

#### 2.1 必须属性

| 属性     | 作用     | 安全规范       | 示例              |
| -------- | -------- | -------------- | ----------------- |
| `href`   | 链接目标 | 必须有效 URL   | `href="/about"`   |
| `target` | 打开方式 | 外链需谨慎使用 | `target="_blank"` |

#### 2.2 关键安全实践

- 外链必须添加 `rel="noopener noreferrer"` ：
- 防止 tabnabbing 攻击
- 避免性能问题（新页面无法访问原页面 window 对象）

#### 2.3 语义化链接类型

| 场景     | 正确写法                                     | 错误写法                          |
| -------- | -------------------------------------------- | --------------------------------- |
| 内部导航 | `<a href="/contact">联系我</a>`              | `<div onclick="...">联系我</div>` |
| 下载文件 | `<a href="resume.pdf" download>下载简历</a>` | 无 download 属性                  |
| 邮箱链接 | `<a href="mailto:me@example.com">发邮件</a>` | 纯文本邮箱                        |



### 三、音视频基础嵌入

---

#### 3.1 视频 `<video>` 

```html
<video controls width="600">
  <source src="demo.mp4" type="video/mp4">
  <source src="demo.webm" type="video/webm">
  您的浏览器不支持视频播放。
</video>
```

- controls：显示播放控件（必须）
- 多 `<source>`：提供备用格式（MP4/WebM）

#### 3.2 音频 `<audio>`

```html
<audio controls>
  <source src="music.mp3" type="audio/mpeg">
  您的浏览器不支持音频播放。
</audio>
```

#### 4.2 关键验证点

- 图片验证：
  - 检查所有 `<img>` 是否有 `alt` 属性
  - 在开发者工具中禁用图片 → 确认 `alt` 文本显示
- 链接验证：
  - 点击导航链接是否跳转（即使404也应触发跳转）
  - 检查外链（GitHub）是否包含 `rel="noopener noreferrer"`
  - 邮箱链接点击是否调用邮件客户端
- 媒体验证：
  - 视频是否显示播放控件
  - 音频/视频在低网速下是否有备用提示文本
- 无障碍测试：
  - 用WAVE插件检查：
    - 无“缺少alt文本”错误
    - 链接有明确目的（非“点击这里”）



### 五、关键建议

---

#### 5.1 alt 文本编写原则

- 装饰性图片：`alt=""`（如分隔线）
- 信息性图片：描述内容（如 `alt="React组件树结构图"`）
- 功能性图片：说明功能（如 `alt="搜索按钮"`）

#### 5.2 链接安全黄金法则

- 只要使用 `target="_blank"`，**必须**加 `rel="noopener noreferrer"`
- 内部链接不要加 `target="_blank"`（破坏用户预期）

#### 5.3 媒体文件最佳实践

- 用占位链接（如 placeholder.com / Google示例视频）
- 真实项目需考虑：
  - 图片压缩（WebP格式）
  - 视频托管（避免直接放MP4到服务器）
