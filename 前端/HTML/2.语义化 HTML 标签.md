>学习目标：
>
>- 理解 HTML 语义化的概念和重要性
>- 掌握常用语义化结构标签的使用场景
>- 能用语义化标签重构第一天的页面，提升代码可读性和 SEO 友好度



### 一、什么是语义化 HTML

---

#### 1.1 定义

使用具有明确含义的 HTML 标签来描述内容的结构和角色，而非仅用样式和布局。

#### 1.2 为什么重要

- **可访问性**：屏幕阅读器能准确理解页面结构（如“这是导航区域”）
- **SEO 优化**：搜索引擎更容易抓取关键内容（如 `<article>` 比 `<div>`）权重更高
- **代码可维护性**：开发者一眼看懂页面逻辑结构

#### 1.3 对比示例

```html
<!-- 非语义化 -->
<div id="top">...</div>
<div class="nav">...</div>
<div class="content">...</div>

<!-- 语义化 -->
<header>...</header>
<nav>...</nav>
<main>...</main>
```



### 二、核心语义化标签详解

---

| 标签        | 作用                   | 使用场景                     | 注意事项                               |
| ----------- | ---------------------- | ---------------------------- | -------------------------------------- |
| `<header>`  | 页面或区块的头部       | 网站 Logo、主导航、表人体组  | 可在页面中多次使用（如文章头部）       |
| `<nav>`     | 导航链接区域           | 主菜单、分页、目录           | 仅用于主要导航，非所有链接都需包裹     |
| `<main>`    | 页面主要内容           | 文章、产品列表、核心功能区   | **整个页面只能有一个**，且不能是子元素 |
| `<article>` | 独立完整的内容单元     | 博客文章、新闻、评论         | 可独立存在（如RSS订阅内容）            |
| `<section>` | 主题相关的区块         | 章节、标签页、功能模块       | 需有明确主题，通常带标题（h1-h6）      |
| `<aside>`   | 与主内容间接相关的内容 | 侧边栏、广告、相关链接       | 不是主要内容但提供补充信息             |
| `<footer>`  | 页面或区块的尾部       | 版权信息、联系方式、相关链接 | 可在页面/文章/区块中多次使用           |

>关键区别：
>
>- `<article>` vs `<section>`：
>  - 如果内容能单独作为一篇文档（如博客），用 `<article>`
>  - 如果只是内容分组（如“技术特点”章节），用 `<section>`
>- `<main>` vs `<article>`：
>  - `<main>` 是整个页面的核心容器，`<article>` 是其中的具体内容单元。



### 三、语义化标签嵌套规则

---

#### 3.1 正确结构示例

```html
<body>
  <header>网站头部</header>
  <nav>主导航</nav>
  <main>
    <article>文章1</article>
    <article>文章2</article>
    <aside>侧边栏</aside>
  </main>
  <footer>页脚</footer>
</body>
```

#### 3.2 禁止嵌套

- ❌ `<header>`内不能包含`<main>`
- ❌ `<footer>`不能作为`<main>`的父元素



### 四、实践任务

---

#### 4.1 步骤 1：分析页面

```html
<body>
    <!-- 页面内容 -->
    <h1>HTML 结构</h1>
    <p>第一个网页</p>
    <img src="../img/image.jpg" alt="示例图片">
</body>
```

问题：所有内容都在`<body>`下平铺，无结构层次。

#### 4.2 用语义化标签重构

将内容按照逻辑拆分并包裹：

```html
<body>
    <!-- 网站头部 -->
    <header>
        <h1>Semantic Tags</h1>
    </header>

    <!-- 主要内容 -->
    <main>
        <article>
            <p>第一个网页</p>
            <img src="../img/image.jpg" alt="示例图片">
        </article>
    </main>

    <!-- 页脚 -->
    <footer>
        <p>Copyright &copy; 2025 chance</p>
    </footer>
</body>
```

#### 4.3 验证语义化效果

- 浏览器检查：

  - 打开开发者工具 → Elements面板

  - 观察标签颜色（Chrome会高亮语义化标签）

- 无障碍测试：

  - 安装浏览器插件 **WAVE**（Web Accessibility Evaluation Tool）

  - 检查是否识别出`<header>`、`<main>`等区域

- 对比实验：
  - 临时将`<main>`改回`<div>`，观察WAVE报告变化



### 五、关键建议

---

- 不要过度语义化：

  - ❌ 错误：为每个段落加`<section>`
  - ✅ 正确：仅对有明确主题的区块使用`<section>`

- 兼容性注意：

  - IE8及以下不支持新语义标签（现代开发无需考虑）

  - 如需兼容旧浏览器，可添加CSS重置：

    ```css
    header, nav, main, article, aside, footer {
        display: block;
    }
    ```
