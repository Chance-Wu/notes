### 一、图文组合标签：`<figure>` 与 `<figcaption>`

#### 1.1 语义定位

- 适用场景：独立于正文的内容性插图（如图表、照片、代码示例、视频）
- 不适用场景：纯装饰图片、页面 UI 图标（仍用普通 `<img>`）

#### 1.2 结构规范

```html
<figure>
  <img src="chart.jpg" alt="2025年Q3用户增长趋势">
  <figcaption>图1：用户季度增长数据（来源：内部统计）</figcaption>
</figure>
```

#### 1.3 关键规则

- 正确：`<figcaption>` 必须是 `<figure>` 的**第一个或最后一个子元素**
- 错误：可包含多种媒体：`<img>`、`<video>`、`<code>`、`<pre>` 等
- 错误：将页面Logo放入 `<figure>`（非独立内容单元）

#### 1.4 与普通 `<img>` 的本质区别

| 场景           | 标签选择           | 原因                           |
| -------------- | ------------------ | ------------------------------ |
| 博客中的示意图 | `<figure>`         | 图片是内容的一部分，需独立说明 |
| 页面背景装饰图 | `<img>` + `alt=""` | 无信息价值，仅为视觉效果       |
| 文章内嵌小图标 | `<img>`            | 非独立内容单元                 |



### 二、图形绘制标签：`<canvas>` 与 `<svg>`

---

#### 2.1 `<canvas>` 特点

- 位图绘制：通过 JavaScript 动态绘制像素（如游戏、数据可视化）

- 无语义：内容对搜索引擎/屏幕阅读器不可见

- 基础用法：

  ```html
  <canvas id="myChart" width="400" height="200">
    您的浏览器不支持 Canvas。
  </canvas>
  <script>
    const ctx = document.getElementById('myChart').getContext('2d');
    ctx.fillStyle = 'blue';
    ctx.fillRect(10, 10, 100, 100);
  </script>
  ```

#### 2.2 `<svg>` 特点

- 矢量图标：XML 格式描述图形（如图标、Logo、简单图标）

- 自带语义：可被搜索引擎索引，支持 ARIA 属性

- 内联用法：

  ```html
  <svg width="100" height="100" aria-label="圆形图标">
    <circle cx="50" cy="50" r="40" fill="green" />
  </svg>
  ```

#### 2.3 选择指南

| 需求                   | 推荐方案                  |
| ---------------------- | ------------------------- |
| 动态交互图形（如动画） | `<canvas>`                |
| 静态图标/Logo          | `<svg>`（内联或外部文件） |
| 响应式图表             | `<svg>`（矢量缩放无损）   |
| 复杂游戏画面           | `<canvas>`                |



### 三、HTML5 语义扩展全景

---

#### 3.1 其他 HTML5 标签

- `<details>`/`<summary>`：可折叠内容区块

- `<figure>` 结构验证：

- Canvas 兼容性：

  - 现代浏览器应显示绿色矩形



### 五、关键建议

---

#### 5.1 `<figure>` 使用红线

- 错误：不要用于页面装饰性图片（如背景图、分割线）
- 正确：仅用于可独立存在的内容单元（即使脱离正文也有意义）

#### 5.2 Canvas vs SVG 选择口诀

静态用 SVG，动态用 Canvas；

要 SEO 选 SVG，要性能选 Canvas。
