### 一、AI 到底在干嘛

---

一句话：

>AI = 用历史数据，训练一个模型，让它对新数据做判断。

拆开来看：

```text
历史数据 -> 学规律 -> 得到模型
新数据 -> 丢进模型 -> 得到结果
```

现在可以吧 AI 想成一个：**超级复杂的经验函数**。



### 二、数据不是“原始世界”，而是“被翻译过的世界”

---

机器看不懂现实。

- 看不懂“这是一只猫”
- 看不懂“这个用户很活跃”

一切都要变成数字。

比如：

| 现实     | 机器看到的 |
| -------- | ---------- |
| 年龄     | 28         |
| 性别     | 0/1        |
| 点击次数 | 37         |
| 文本     | 一串向量   |

 结论非常重要：

>**AI 的上限，首先被“你给它的数据形态”限制。**



### 三、特征（Feature）=AI 眼中的世界

---

**特征 = 你告诉 AI：哪些信息是重要的。**

举个例子：
 预测一个用户是否会下单。

可能的特征：

- 最近 7 天浏览次数
- 是否加过购物车
- 是否领过优惠券
- 上一次购买时间

会发现：**这是业务理解，不是算法理解。**

> 做 AI 的人，70% 的时间在想“特征选什么”



### 四、模型不是规则，而是“经验浓缩包”

---

模型不是 if-else

不会看到：

```java
if (age > 30 && click > 10) then buy = true
```

而是：

```text
一堆参数 + 计算结构
```

这些参数来自：

- 数据
- 训练过程
- 误差调整

>**模型 = 数据里“统计规律”的压缩表示**



### 五、训练在干什么

---

训练的本质只有一件事：

>不断犯错 -> 不断修正

流程：

1. 模型先随便猜
2. 和真实的答案对比
3. 算“错多少”
4. 调整参数
5. 重复几千/几百万次

这就是为什么：

- 数据越多 -> 越容易学对
- 噪声越大 -> 学得越歪



### 六、预测其实很“朴素”

---

预测阶段：

```text
新数据 -> 特征化 -> 丢进模型 -> 出一个结果
```

这个结果可能是：

- 一个概率（0.87）
- 一个分类（是 / 否）
- 一个数值（价格）

**预测本身不“智能”，只是套用经验。**



### 七、你现在应该有的“正确错觉”

---

到这里，你应该开始有这些感觉：

- [x] AI 很强，但不神秘
- [x] AI 更像统计，而不是思考
- [x] 数据和业务理解 > 算法花活
- [x] 模型只是“被训练出来的工具”



### 八、一个程序员视角的类别

---

如果写过代码生成器 / ORM / 规则引擎：

- 传统系统：
  人写规则
- AI 系统：
  数据“写规则”

你只是把：

```
规则维护成本
```

换成了：

```
数据 & 训练成本
```
