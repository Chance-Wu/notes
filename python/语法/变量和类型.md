java 中，每个变量在编译时就必须确定身份（例如 `int count = 10;`）。而在 python 中，它是动态类型的：变量更像是一个标签，可以贴在任何对象上。你不需要声明类型，直接赋值即可：`count=10`。

>python 抛弃了 java 繁琐的“大括号+分号”组合，转而使用强制缩进（通常是 4 个空格）来定义代码快。



### 变量与数据类型

```python
# 字符串
name = "小明"

# 数字
price = 99.9
quantity = 10

# 列表（可变，有序）
tools = ["计算器", "日志分析器", "文件整理器"]

# 字典（键值对，像小型数据库）
user = {
    "name": "小明",
    "tools_used": ["计算器", "日志分析器"]
}
```

>使用 `type()` 查看变量类型。
>
>```python
>print(type(tools))  # <class 'list'>
>```

#### 动态指针数组

在 Java 中，`ArrayList<Integer>` 在内存中会尽量保持连续的原始类型或特定类型的引用。而 Python 的列表本质上是一个**动态指针数组**。列表里存的不是对象本身，而是指向这些对象的**引用（指针）**。因为 Python 中“一切皆对象”，所以列表只需要存储一串地址，至于地址指向的是 `int` 还是 `str`，列表并不关心。

| 特性     | 优点（Pro）                                        | 缺点（Cons）                                                 |
| -------- | -------------------------------------------------- | ------------------------------------------------------------ |
| 灵活性   | 处理 JSON或异构数据非常方便，无需定义复杂的 DTO 类 | 缺乏**类型安全**。容易在运行时抛出 `TypeError`（例如尝试对字符串做加法）。 |
| 开发速度 | 减少了样板代码（Boilerplate），写脚本和原型极快。  | **重构困难**。没有 Java 编译器那种强大的静态检查，大型项目维护成本高。 |
